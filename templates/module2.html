Why Do We Need DNS?

In our last module, we took a long tour of how data travels across the internet using IP addresses, packets, and the moderation of routers and ISPs. We also talked about the idea that your computer needs an IP address to find a website. But here's the most important part of this ordeal: have you ever tried to remember a bunch of random numbers, like 172.217.160.142 or 2001:0db8:85a3:0000:0000:8a2e:0370:7334? Probably not. That's where our main course, DNS, comes in.

The Problem with IP Addresses

Imagine every time you wanted to call a friend, you had to remember a long, complex phone number made up of random digits. You probably haven't ever had to do that. Not only that, but if your friend moved and got a new number, you'd have to find out their new number and update your pretty cramped mental directory. That sounds exhausting enough. Now, scale that up to the entire internet!

Every website you visit, every online service you use, and every email you send connects to a server that has a unique IP address. If we had to manually type in IP addresses for websites like 142.250.190.46 (Google's IP address on one of their servers) instead of google.com, the internet would be incredibly difficult to use. You'd probably be using things like Internet Phonebooks, which map IP Addresses to websites. Now that's what I would call frustrating.

Here are the main problems with relying solely on IP addresses for human interaction:

  • Numbers are much harder for humans to memorize than meaningful names.
  • An IP address like 104.26.15.118 tells you nothing about the content or use of the website it refers to.
  • Server IP addresses can change. Websites might move to different hosting providers, or companies might reconfigure their networks. If an IP address changes, and you've memorized it, your "link" to that website is broken.
  • Large websites often use multiple servers with different IP addresses to handle heavy traffic or provide backup in case of a server failing. How would you pick which IP address to use, or know when to switch? This is another major issue that must be solved, as massive datacentres with multiple servers are built all the time.

This is where the genius of the DNS, or Domain Name System, truly saves our lives, and our brain's storage capacity.

Human-Friendly vs. Machine-Friendly Names

Our brains are fantastic at remembering words, phrases, and patterns. facebook.com, wikipedia.org, amazon.in and google.com are easy to remember, type, share and guess. (Try guessing some probable website names. You'll probably land on one.) They often give us a clue about the website's content. These are human-friendly names, also known as domain names.

On the other hand, computers and network devices (like routers) use numbers. They are incredibly efficient at processing and routing data based on the format of an IP address. These are machine-friendly names (IP addresses).

The challenge that we have to tackle as of now is bridging this gap. We need a system that allows us to use easy-to-remember domain names while still giving computers the numerical IP addresses they need to actually connect to.

The Phonebook of the Internet

The best way to understand the primary, essential function of DNS is to think of it as the internet's giant, distributed phonebook. (Hey, there IS an internet phonebook!)

If you wanted to call the "Papa Tomato's Pizza Palace," back in the 'good ol' days' of phonebooks, you'd look up "Papa Tomato's Pizza Palace" in the phonebook, find their name, and next to it, you'd see their phone number. You'd then dial the number.

DNS works exactly the same way:

  • You know the "name" of the website you want to visit (e.g., www.google.com).
  • Your computer asks the DNS "phonebook" system, "What's the IP address for www.google.com?"
  • The DNS system looks it up and replies, "It's 142.250.190.46."
  • Your computer then takes that IP address and uses it to connect to the server hosting www.google.com.

This analogy is powerful because it highlights the core task of DNS. That task is translating human-readable domain names into machine-readable IP addresses. An honorable profession.

The Core Function of DNS: Name Resolution

The process of translating a domain name into an IP address is called name resolution. This is the most crucial role of the Domain Name System.

Every time you click a link or type in a site address, a DNS lookup (or "resolution") occurs in the background. It's happening constantly. Millions of times per second across the globe, DNS resolutions take place so quickly you don't even notice it.

Without name resolution, your browser wouldn't know where to send its request for www.google.com. It's like trying to send a letter without knowing the recipient's street address.

Beyond simply giving back an IP address, DNS also provides:

  • Redundancy. If a server's IP address changes, the DNS entry can be updated, and the rest of the internet will quickly learn the new address. Users don't need to know or do anything at all on their end.
  • Load balancing is also graciously provided. For very popular websites, DNS can provide different IP addresses to different users across the globe, directing traffic to various servers to prevent any one server from getting overloaded.
  • Mail Routing! DNS is wonderfully versatile. It also tells your email client which server handles email for a specific domain (e.g., google.com's mail server).
  • Service Discovery: It can point to servers for various services beyond web pages and email.

In short, without DNS, the internet would transform into an obscure, difficult to operate network primarily for researchers and engineers who could manage direct IP connections. DNS is what makes the internet accessible and manageable for billions of people worldwide.
