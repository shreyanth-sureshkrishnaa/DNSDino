Glossary

A Record (Address Record)

A fundamental DNS record type that maps a domain name to an IPv4 address (e.g., example.com to 192.0.2.1). It's how your browser finds the numerical address of a website.

AAAA Record (Quad-A Record)

Similar to an A record, but it maps a domain name to an IPv6 address (e.g., example.com to 2001:0db8::1). IPv6 is the newer internet protocol designed to handle more addresses.

AA Flag (Authoritative Answer Flag)

A flag in a DNS response that indicates the answering name server is authoritative for the domain in question. This means the server holds the original, official records for that domain.

Anycast

A network addressing and routing method where data is routed to the "nearest" or "best" destination among a group of potential receivers, all sharing the same IP address. This is often used for DNS servers to improve performance and resilience.

Authoritative Name Server

A DNS server that holds the original, definitive DNS records for a particular domain. When you look up a website, a recursive resolver eventually queries an authoritative name server to get the correct information.

Bandwidth

The maximum rate at which data can be transferred over a network connection, typically measured in bits per second (bps). It's essentially how much data "pipe" you have.

BGP (Border Gateway Protocol)

A routing protocol used to exchange routing information between autonomous systems (large networks like ISPs) on the internet. It's the "postal service" of the internet, directing traffic.

Cache

A temporary storage location for data that is frequently accessed. In DNS, resolvers cache query results to speed up future lookups.

CDN (Content Delivery Network)

A geographically distributed network of proxy servers and their data centers. The goal is to provide high availability and performance by distributing the service spatially relative to end-users. Think of it as having copies of website content closer to you.

CNAME Record (Canonical Name Record)

A DNS record that maps one domain name to another (an alias). For example, www.example.com might be a CNAME for example.com, meaning they point to the same underlying resource.

DNS (Domain Name System)

The internet's phonebook. It translates human-readable domain names (like google.com) into machine-readable IP addresses (like 172.217.160.142).

DNS Resolver (Recursive Resolver)

A server that receives DNS queries from client applications (like your web browser) and performs the necessary lookups to find the corresponding IP address. It recursively queries other DNS servers until it gets an answer.

DNSSEC (Domain Name System Security Extensions)

A suite of security extensions that add cryptographic signatures to DNS data, helping to ensure the authenticity and integrity of DNS responses and protect against certain types of attacks.

Domain Name

A human-friendly name used to identify a website or resource on the internet (e.g., example.com).

Firewall

A network security system that monitors and controls incoming and outgoing network traffic based on predetermined security rules. It acts as a barrier between a trusted internal network and untrusted external networks (like the internet).

FQDN (Fully Qualified Domain Name)

A complete domain name that specifies its exact location in the DNS hierarchy, including the hostname and all parent domains (e.g., www.example.com.). The trailing dot signifies the root of the DNS tree.

Gateway

A network device that connects two different networks and translates protocols between them. Often, your home router acts as a gateway to the internet.

ICMP (Internet Control Message Protocol)

A network layer protocol used by network devices to send error messages and operational information, such as whether a requested service is available or if a host or router can be reached. Ping uses ICMP.

IP Address (Internet Protocol Address)

A unique numerical label assigned to each device connected to a computer network that uses the Internet Protocol for communication. It's like a street address for your device on the internet.

ISP (Internet Service Provider)

A company that provides individuals and organizations access to the internet and other related services.

Latency

The delay before a transfer of data begins following an instruction for its transfer. Often referred to as "lag" in gaming.

Load Balancer

A device or software that distributes network traffic across multiple servers to ensure no single server becomes overwhelmed. This improves responsiveness and availability.

MX Record (Mail Exchange Record)

A DNS record that specifies the mail server responsible for accepting email messages on behalf of a domain name. It tells email senders where to deliver mail for your domain.

Name Server

A server that stores DNS records and responds to DNS queries.

NXT Record (Next Record)

An older DNSSEC record type (now deprecated by NSEC and NSEC3) that was used to provide "proof of non-existence" for a DNS record, helping to secure against certain types of attacks.

Packet

A small unit of data transmitted over a network. Data is broken down into packets for efficient transmission and then reassembled at the destination.

Ping

A network utility used to test the reachability of a host on an Internet Protocol (IP) network and to measure the round-trip time for messages sent from the originating host to a destination computer.

Protocol

A set of rules that governs how data is formatted and transmitted over a network. Examples include HTTP, TCP, and IP.

Recursive Query

A type of DNS query where the DNS resolver is asked to provide a complete answer to the query, even if it has to query other DNS servers to get it.

Root Name Server

The highest level of the DNS hierarchy. There are 13 logical root name servers worldwide, which direct queries to the appropriate top-level domain (TLD) name servers.

Router

A networking device that forwards data packets between computer networks. Routers perform the traffic directing functions on the Internet.

Subnet Mask

A 32-bit number that distinguishes the network address and the host address within an IP address. It helps define the boundaries of a local network.

TCP (Transmission Control Protocol)

A core protocol of the internet protocol suite. It provides reliable, ordered, and error-checked delivery of a stream of bytes between applications running on hosts communicating via an IP network.

TLD (Top-Level Domain)

The last segment of a domain name, located after the final dot (e.g., .com, .org, .net, .gov).

TTL (Time To Live)

A value in a DNS record that tells a DNS resolver how long it should cache a record before requesting a fresh copy from the authoritative name server. A shorter TTL means changes propagate faster.

UDP (User Datagram Protocol)

A simpler, faster, and connectionless communication protocol compared to TCP. It's used when speed is more important than guaranteed delivery (e.g., streaming video, online gaming).

URL (Uniform Resource Locator)

A specific type of URI that provides a means of locating a resource on the internet, typically a web page (e.g., https://www.example.com/page).

VPN (Virtual Private Network)

A technology that creates a secure, encrypted connection over a less secure network, such as the internet. It allows users to send and receive data as if their computing devices were directly connected to the private network.

Wi-Fi

A wireless networking technology that allows devices to connect to a network and the internet using radio waves.
